# 当前项目说明与使用指南

## 1. 项目简介

本项目是一个基于多模态情绪感知与情感生成的虚拟陪伴交互系统原型，当前已经具备从用户输入到陪伴式回应的完整演示链路。

系统当前支持：

- 语音输入与语音识别
- 文本情绪识别
- 语音情绪识别
- 摄像头表情识别
- 多模态情绪融合
- 情感化文本回复生成
- 情感语音生成与播放
- 页面内连续对话上下文记忆


## 2. 当前技术栈

### 前端

- 原生 `HTML / CSS / JavaScript`
- 浏览器 `MediaDevices API`：麦克风与摄像头采集
- 浏览器 `AudioContext`：录音与 WAV 编码
- 浏览器 `speechSynthesis`：仅作为极端情况下的兜底朗读能力

### 后端

- `Python`
- `Flask`
- `flask-cors`
- `python-dotenv`

### 深度学习与模型推理

- `PyTorch`
- `transformers`
- `onnxruntime`
- `OpenCV`
- `librosa`
- `soundfile`

### 当前已接入的模型 / 服务

- 语音识别：`DashScope / Qwen ASR`
- 文本情绪识别：`Qwen API`
- 语音情绪识别：`superb/wav2vec2-base-superb-er`
- 表情识别：`FERPlus ONNX`
- 对话生成：`Qwen API`
- 情感语音生成：`Qwen3-TTS`


## 3. 当前具备的功能

### 3.1 交互模块

- 点击录音按钮后可采集麦克风音频
- 可开启摄像头实时采样
- 摄像头区域会显示当前表情识别结果
- 同一页面内支持连续多轮对话，不会每轮都从头开始

### 3.2 情感识别模块

- 文本情绪识别：根据语音识别结果调用 Qwen 做情绪分类
- 语音情绪识别：使用 `wav2vec2` 情绪分类模型做基线推理
- 表情识别：使用 ONNX FERPlus 模型识别人脸表情
- 多模态融合：综合文本、语音、表情结果输出最终情绪

### 3.3 回应模块

- 根据当前情绪和最近几轮上下文生成陪伴式回答
- 回答支持短回复与较深入回复，不再固定只有一句话
- 当前页面内对话连续，刷新页面后重置上下文
- 支持播放 Qwen3-TTS 生成的情感语音


## 4. 当前性能说明（原型阶段）

以下数据为当前开发环境下的典型观察值，不代表严格基准测试结果，实际速度会受到网络、电脑性能、模型首次加载和 API 波动影响。

### 4.1 页面交互体验

- 前端页面加载：较快，通常为本地静态资源加载
- 摄像头实时表情识别：约每 `1.2s` 更新一次
- 同一页面内上下文记忆：即时生效

### 4.2 后端链路性能（观察值）

- 文本情绪识别：通常较快，主要受 Qwen API 网络耗时影响
- 对话生成：通常在 `1s` 左右到数秒内完成
- Qwen3-TTS：常见耗时约 `2s - 5s`
- 首次调用模型时可能更慢，后续调用通常会更稳定

### 4.3 当前已知限制

- 语音情绪识别模型目前采用英文情绪基线模型，中文场景精度仍有提升空间
- Qwen3-TTS 在网络不稳定时会出现延迟变大
- 当前 TTS 仍是“合成完成后播放”，不是完全流式播放
- ASR / LLM / TTS 都依赖外部 API 时，整体响应会受网络影响

## 6. 运行环境要求

建议环境：

- Windows
- Python 3.11（项目当前推荐使用 `.venv311`）
- 可联网（因为当前接入了 DashScope / Qwen API）
- 浏览器支持麦克风与摄像头权限

## 9. 配置说明

项目根目录使用 `.env` 配置运行参数。常用配置如下：

```env
APP_HOST=0.0.0.0
APP_PORT=5000
APP_DEBUG=false
USE_MOCK_SERVICES=false

ASR_PROVIDER=qwen_dashscope
TEXT_EMOTION_PROVIDER=qwen_dashscope
TEXT_EMOTION_MODEL=qwen-turbo
SPEECH_EMOTION_PROVIDER=wav2vec2_superb
SPEECH_EMOTION_MODEL=superb/wav2vec2-base-superb-er

DASHSCOPE_API_KEY=你的API Key
DASHSCOPE_ASR_MODEL=paraformer-realtime-v2

LLM_PROVIDER=qwen_dashscope
LLM_MODEL=qwen-turbo

TTS_PROVIDER=qwen_dashscope
DASHSCOPE_TTS_MODEL=qwen3-tts-instruct-flash-realtime
DASHSCOPE_TTS_VOICE=Cherry
DASHSCOPE_TTS_URL=wss://dashscope.aliyuncs.com/api-ws/v1/realtime
```


## 10. 使用步骤

### 基本使用流程

1. 启动后端
2. 启动前端
3. 打开浏览器访问 `http://127.0.0.1:8000`
4. 允许麦克风和摄像头权限
5. 如需实时表情识别，点击“开启实时表情识别”
6. 点击录音按钮，说出当前状态
7. 等待系统完成识别、融合、生成回应和语音播放

### 连续对话说明

- 同一页面内最近几轮对话会自动保留，生成回复时会参考上下文
- 如果刷新页面，对话上下文会被清空并重新开始


## 11. 常见问题

### 11.1 前端打不开

- 确认是否已启动 `python -m http.server 8000`
- 确认 `8000` 端口没有被其他程序占用

### 11.2 后端打不开

- 确认是否使用了 `.venv311\Scripts\python.exe`
- 确认 `5000` 端口没有被占用
- 查看 `logs/backend-runtime.log`

### 11.3 摄像头没有画面

- 检查浏览器是否允许摄像头权限
- 检查是否有其他软件占用摄像头

### 11.4 语音播放很慢或失败

- 当前 TTS 依赖 Qwen3-TTS API，速度受网络影响
- 如果失败，优先检查网络和 DashScope Key 是否可用

### 11.5 对话没有连续性

- 只有在同一页面内连续对话才会保留上下文
- 刷新页面后会重置
